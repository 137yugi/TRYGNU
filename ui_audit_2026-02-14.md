# UI 最適化 / 画面棚卸（2026-02-14）

## 0. 結論
- ゲーム体験画面は「ゲーム画面（Canvas）＋インゲームメニューボタン」を保てており、
  メニュー系の操作をボタン1つに集約できている。
- ただし、DOM・CSS・JS上には“現在見えていないUI”が複数残っている。
  このままでも動作は壊れないが、保守性の観点で整理余地あり。

## 1) 画面で表示しているUI（人が使う操作面）

### A. ゲーム画面（常時表示）
- Canvas（`#gameCanvas`）
- スタートボタン（`#startBtn`）
- メニューボタン（`#menuFloatingBtn`）

### B. メニュー画面（`#menuModal` でフルスクリーン）
- キャラクター
  - 生成（`#rollCharBtn`）
  - ジョブ/武器選択（`#jobSelect`, `#weaponSelect`）
  - 生成情報表示（`#charNameVal`, `#charSpecVal`）
- 操作
  - 音（`#audioBtn`）
  - ギフト（`#gift100Btn`, `#gift500Btn`, `#gift1000Btn`）
  - ライブ連動（`#streamHookBtn`, `#streamHookStatus`）
  - +20C購入（`#buyCreditBtn`）
  - 表示調整（`#systemTextBtn`, `#systemFlashBtn`, `#systemShakeBtn`）
  - 用語集（`#openGlossaryBtn`）
- 補助数値（売上）
  - `#grossVal`, `#feeVal`

### C. 用語集
- `#glossaryModal`, `#glossaryList`, `#closeGlossaryBtn`

### D. 一時停止モーダル（発生条件付き）
- レベルアップ: `#levelModal`
- 変異: `#mutationModal`
- 装備比較: `#pickupModal`

## 2) 画面に表示されていないのに存在しているUI（未使用・残存）

### 2-1. CSSで常時非表示
- `#menuModal .level-card`（メニュー内ではレベル/変異/装備比較の重複表示を抑制するため）

### 2-2. HTMLから削除済み（コード上は参照が残る）
- `dropList`, `pickupTrail`, `logBox`, `rankingBody`, `hint`
- 対応ID（削除済み要素）: `dropList`, `pickupTrail`, `logBox`, `rankingBody`
- HUD系
  - Wave / Level / HP / 敵速度 / Fury / 時間 / Credits / Gift / Legendary / スコア / XP / Intent / Objective / ObjectiveTimer
  - 対応ID: `waveVal`, `levelVal`, `hpVal`, `enemySpeedVal`, `furyVal`, `timeVal`, `glitchTimeVal`, `creditVal`, `giftVal`, `legendVal`, `scorePreviewVal`, `xpFill`, `xpTextVal`, `intentVal`, `objectiveVal`, `objectiveTimerVal`
  - `stats` コンテナ
- インベントリ・装備系（大きく削除）
  - `#gearSlots`, `#itemList`, `#itemDetailTitle`, `#itemCompareDelta`, `#itemQuickMeta`, `#itemDetailStats`
  - `#equipItemBtn`, `#salvageItemBtn`, `#unequipSlotBtn`
  - `#affixList`, `#runeList`, `#loadoutInfo`, `#setBonusVal`, `#gearHint`
  - 旧ルーン/アフィックス操作はv1対象外。現行は装備比較（`#pickupModal`, `#pickupKeepBtn`, `#pickupDiscardBtn`）で確認する。
- 機能ボタンはキーボード残置のまま UI だけ削除
  - `#glitchBtn`（固定ON機能自体は残っている）

### 2-3. 条件表示のため存在するが、普段は見えない
- レベルアップ/変異/装備比較モーダル本体（各種イベント時のみ）
- メニュー系オーバーレイの開閉で `pointer-events` を遮断するため、ゲーム画面入力は一時停止
- 画面切替キーで表示中継続される HUD/設定系トグル (`h / t / v / j / f`)

### 2-4. いまは UI から触れないが内部ロジックとして残存
- ルーン/アフィックス系の選択・刻印・再鍛造
  - `ui.runeList`, `ui.affixList`, `ui.itemList`, `ui.affixLabHint`, `affix*Btn`
- 装備/インベントリ系の比較・破棄・装着フロー
  - `ui.gearSlots`, `ui.itemDetail*`, `ui.equipItemBtn`, `ui.salvageItemBtn`, `ui.unequipSlotBtn`, `ui.setBonusVal`, `ui.loadoutInfo`, `ui.gearHint`
- HUD系数値や統計描画処理
  - `ui.wave`, `ui.level`, `ui.hp`, `ui.enemySpeed`, `ui.fury`, `ui.time`, `ui.glitchTime`, `ui.credit` `ui.gift`, `ui.legendary`, `ui.scorePreview`, `ui.xpFill`, `ui.xpText`, `ui.intent`, `ui.objective`, `ui.objectiveTimer`, `ui.stats`
- トップ画面で削除対象だった外部指標
  - ドロップ履歴・ログ・ランキング
- グリッチ固定UI
  - `ui.glitchBtn` 本体の要素は削除、`a` キー/内部フラグ更新の処理は残存

## 3) 機能一覧（人間向けの操作単位）

### 常時利用
- ゲーム起動/開始/再開: `startBtn`, `mobileStartBtn`, Enterキー/ゲーム内タップ
- 移動: Canvas タップ/ドラッグ、方向キー/WASD
- 画面外入力遮断（`menu`ボタン/メニュー開閉）
- 画面内一時停止: メニューを開く、レベル/変異/比較モーダル
- 全画面メニュー表示/閉じる
  - `menuFloatingBtn`, `closeMenuBtn`, `Esc`/`M` キー
- 主要画面制御
  - 音ON/OFF、フラッシュ/シェイク ON/OFF、テキスト量切替
  - フルスクリーン切替

### メニュー側
- キャラクター生成/選択
- ギフト・音量関連
- 購入/経済情報
- ライブ連動の ON/OFF
- 表示設定（テキスト/フラッシュ/シェイク）
- 用語集の閲覧

### 一時イベント側面（特定条件）
- レベルアップ 3択 + リロール（`rerollSkillBtn`）
- 変異2択
- ドロップ装備比較（装備/破棄）
- ボス/ミニボス進行
- ギフトイベント進行
- 端末入力イベント取り込み（`#terminalChannelInput`, `#connectTikTokBtn`, `#terminalTestEventBtn`）

### 非表示だが内部には存在する側面
- インベントリ管理 + ルーン/アフィックス系ガチャ・再鍛造
- リアルタイムヒント/ログ/ランキング記録の更新
- Glitch 固定ボタン機能（キーボードAキー）
- HUD Compact・表示系トグル（H/T/V/J/F）
- ギフト履歴/リーダーボード更新処理

### 非画面系機能
- 敵ウェーブ・ボス/ミニボス生成、難易度自動調整
- エフェクト/音声（SFX）管理
- 画面描画最適化・リトライ条件処理（失敗/敗北/クリア）
- 端末入力イベント取り込み（BroadcastChannel / postMessage / storage / CustomEvent + キュー処理）
- スコアリング/ランキング保存とカテゴリ分岐（CLEAN / GLITCH）

## 4) 見えている画面と見えていない画面

### 見えるもの
- ゲーム画面: `start`, `menu`, `canvas`
- メニュー: 生成・武器ジョブ選択・ギフト関連・視認ボタン群・用語集
- 変異/レベル/比較はイベント時のみモーダル表示

### 見えないもの
- 画面外HUDの数値群（Wave/HP/Time...）
- インベントリ/ルーン/装備ラボ
- Drop/Log/Ranking エリア（DOM削除済み。処理は `game.js` 側でガード付きのまま保持）
- `glitch`固定トグルUI

## 5) 画面ごとの機能掲載可否

- `game-shell`（ゲーム画面）
  - 直接見える: ゲームCanvas、`startBtn`、`menuFloatingBtn`、一時停止/再開系インジケータ（ボタンラベル）
  - この画面上でのみ常時操作: 移動、タップ開始/解除

- `#menuModal`（フルスクリーン）
  - 常時見える: キャラクター選択、ギフト、配信連動、表示設定、用語集導線
  - メニューを閉じるとゲーム画面へ復帰

- `#levelModal`
  - レベルアップ時のみ: スキル3択、リロール

- `#mutationModal`
  - 変異発生時のみ: スキル2択

- `#pickupModal`
  - 装備ドロップ比較時のみ: 装備/破棄

- `#glossaryModal`
  - 用語集ボタンで表示: 用語辞典

- 非表示（DOM内 or 削除済）
  - 削除済み: ヒント、ドロップ履歴、ログ、ランキング本体
  - DOM内非表示: イベント時モーダル (`level/mutation/pickup`) とメニュー外フレーム系
  - ロジック内のみ: インベントリ/ルーン/HUD統計/古い表示系
